# A-1 任务一 登录安全加固

1. 密码策略（Windows，Linux）1 最小密码长度不少于 8 / 13 / 15 个字符；

2 密码最短使用期限为 10 天。

3 强制密码历史为 5 个密码；

4 密码最长存留期为 45 天；

5. 一分钟内仅允许 4 / 5 次登录失败，超过 4 / 5 次，登录帐号锁定 1 分钟。

6. 密码策略必须同时满足大小写字母、数字、特殊字符。

A-2 本地用户配置（Windows）

1. 在用户登录系统时，应该有 “For authorized users only” 提示信息；

2. 远程用户非活动会话连接超时应小于等于 5 分钟。

3. 设置取得文件或其他对象的所有权，将该权限只指派给 administrators 组；

4. 设置 user1 用户只能在上班时间（周一至周五的 9:00~18:00 可以登录，将 user1 的登录时间配置界面截图；

5. 设置操作系统中的关键目录（system32、hosts、Program Files、Perflogs）的权限为最优状态，即仅允许管理员用户进行读取及运行。

6. 设置不显示上次登录的用户名。

7. 要求登录时不显示用户名；

8. 交互式登录时不显示用户名；

9. 在组策略中只允许管理员账号从网络访问本机；

10. 对服务器进行远程管理安全性 SSL 加固，防止敏感信息泄露被监听；

11. 查找并删除服务器中可能存在的帐号 hacker;

12. 查找并删除服务器中可能存在的后门用户；

同上题，查询全部用户然后删除即可，考虑到隐藏用户这个东西，建议用户与组或者注册表看

13. 密码过期的前 5 天开始提示用户在过期之前更改密码；

14. 要求任何用户在登录到 Windows 前都必须按 CTRL+ALT+DEL；

15. 普通用户进行最小权限管理，对关闭系统仅限管理员帐号。

16. 禁止从远端系统强制关机，将该权限只指派给 administrators 组。

17. 禁用来宾账户，禁止来宾用户访问计算机或访问域的内置账户；

18. 禁止普通用户使用注册表编辑工具以及 Ctrl+Alt+Del；

19. 禁止 SAM 帐户和共享的匿名枚举；

20. 禁止系统在未登录的情况下关闭；

21. 禁止软盘复制并访问所有驱动器和所有文件夹；

22. 禁止自动管理登录；

23. 禁止存储网络身份验证的密码和凭据；

24. 禁止普通用户使用命令提示符；

25. 禁止将 Everyone 权限应用于匿名用户；

26. 在超过登录时间后强制注销。

27. 关闭系统时清除虚拟内存页面文件；

# A-3 服务安全配置（Windows）

3.1 禁用 TCP/IP 上的 NetBIOS 协议，关闭监听的 UDP 137（netbios-ns）、UDP 138（netbios-dgm）以及 TCP 139（netbios-ssn）端口；

然后在防火墙依次找到这三个端口进行关闭

3.2 设置从屏幕保护恢复时需要输入密码，并将屏幕保护自动开启时间设定为五分钟；

3.3 对于远程登录的帐户，设置不活动超过时间 5 分钟自动断开连接。

3.4 在本地策略里配置禁用未登陆前关机；

# A-4 日志安全配置（Windows）

4.1 配置审核登陆，记录内容包括用户登录使用的账户、登录是否成功、登录时间、以及远程登录时间、及用户使用的 IP 地址；

​

4.2 启用本地安全策略中对 Windows 系统的审核策略更改，成功和失败操作都需要审核；

4.3 启用本地安全策略中对 Windows 系统的审核对象访问，成功和失败操作都需要审核。

4.4 启用本地安全策略中对 Windows 系统的审核目录服务访问，仅需要审核失败操作；

4.5 启用本地安全策略中对 Windows 系统的审核特权使用，成功和失败操作都需要审核；

4.6 启用本地安全策略中对 Windows 系统的审核系统事件，成功和失败操作都需要审核。

4.7 启用本地安全策略中对 Windows 系统的审核帐户管理，成功和失败操作都要审核；

4.8 启用本地安全策略中对 Windows 系统的审核进程追踪，仅失败操作需要审核。

4.9 安全日志文件大小至少为 128MB，设置当达到最大的日志大小上限时，覆盖早于 30 天的日志；

4.10 应用日志文件大小至少为 64MB，设置当达到最大的日志大小上限时，覆盖早于 15 天的日志；

4.11 系统日志大小至少为 32MB，设置当达到最大的日志大小上限时，按需要覆盖事件。

# A-5 数据库安全策略

5.1 以普通帐户 mysql 安全运行 mysql 服务，禁止 mysql 以管理员帐号权限运行；

5.2 删除默认数据库 (test)；

5.3 改变默认 mysql 管理员用户为：SuperRoot；

5.4 使用 mysql 内置 MD5 加密函数加密用户 user1 的密码为 (P@ssw0rd1!)。

5.5 赋予 user1 用户对数据库所有表只有 select,insert,delete,update 权限。

对 Web 网站进行 HTTP 重定向 HTTPS 设置，仅使用 HTTPS 协议访问网站（Windows）(注：证书颁发给 test.com 并通过 https://www.test.com 访问 Web 网站)。

# A-7 防火墙策略

1.Windows 系统禁用 445 端口；

2.Windows 系统禁用 23 端口；

以上两个高级防火墙设置禁用就 ok

6.1.Linux 系统使用 iptables 禁用 23 端口；

6.2.Linux 系统使用 iptables 禁止别人 ping 通。

6.3. 禁止本机 ping 任何机器；

6.4. 为防止 Nmap 扫描软件探测到关键信息，设置 iptables 防火墙策略对 3306 号端口进行流量处理；

6.5. 为防止 SSH 服务被暴力枚举，设置 iptables 防火墙策略仅允许 172.16.10.0/24 网段内的主机通过 SSH 连接本机；

6.6. 为防止 Nmap 等扫描软件探测到关键信息，设置 iptables 防火墙策略对 80 号端口进行流量处理；

6.7. 为防御拒绝服务攻击，设置 iptables 防火墙策略对传入的流量进行过滤，限制每分钟允许 3 个包传入，并将瞬间流量设定为一次最多处理 6 个数据包（超过上限的网络数据包将丢弃不予处理）；

6.8. 为防御 IP 碎片攻击，设置 iptables 防火墙策略限制 IP 碎片的数量，仅允许每秒处理 1000 个。

6.9. 为确保安全 Linux 系统禁止所有人通过 ssh 连接除了 172.16.1.1 这个 ip；

6.10. 设置防火墙允许本机转发除 ICMP 协议以外的所有数据包；

6.11. 只允许转发来自 172.16.0.0/24 局域网段的 DNS 解析请求数据包。

6.12. 拒绝 TCP 标志位全部为 1 及全部为 0 的报文访问本机。

6.13. 禁止转发来自 MAC 地址为 29:0E:29:27:65:EF 主机的数据包。

6.14. 在工作时间，即周一到周五的 8:30-18:00，开放本机的 ftp 服务给 192.168.1.0 网络中的主机访问；

6.15. 要求从 ftp 服务的数据下载请求次数每分钟不得超过 5 个；

6.16. 配置 iptables 防火墙过滤规则，以封堵目标网段（172.16.1.0/24），并在两小时后解除封锁。

# A-8 IP 协议安全配置

1. 指定触发 SYN 洪水攻击保护所必须超过的 TCP 连接请求数阈值为 5；

2. 指定处于 SYN_RCVD 状态的 TCP 连接数的阈值为 500；

3. 指定处于至少已发送一次重传的 SYN_RVCD 状态中的 TCP 连接数的阈值为 400。*2

# A-9 中间件服务安全加固 VSFTPD/HTTPD/BIND（Linux）

SSH 服务加固（Linux）

1. 修改 ssh 服务端口为 2222；

2.ssh 禁止 root 用户远程登录；

3. 设置 root 用户的计划任务。每天早上 7:50 自动开启 ssh 服务，22:50 关闭；每周六的 7:30 重新启动 ssh 服务；

4. 修改 SSHD 的 PID 档案存放地。

5. 修改 SSH 连接界面静置时间；

6. 修改登录记录的等级为 INFO;

7. 禁止登陆后显示信息。

VSFTPD

1.vsftpd 禁止匿名用户上传；

2. 置无任何操作的超时时间为 5 分钟；

3. 设置数据连接的超时时间为 2 分钟；

4. 限制本地用户登陆活动范围限制在 home 目录。

5. 设置本地用户创建文件的权限为 022。

6. 设置运行 vsftpd 的非特权系统用户为 pyftp；

7. 匿名用户访问的最大传输速率为 512KB/S；

8. 用户访问的最大传输速率为 1M。

9. 同一客户机 IP 地址允许最大客户端连接数 10；

10. 最大客户端连接数为 100；

11. 限制客户端连接的端口范围在 50000-60000；

HTTPD

1. 更改默认监听端口为 6666；

2. 设置禁止目录浏览；

3. 隐藏 Apache 版本号；

4. 将 Apache 服务降权，用户为 apache，用户组为 www。

BIND

1. 隐藏 bind 版本号；

2. 设置不提供递归服务。

IIS 加固（Windows）

1. 开启 IIS 的日志审计记录 (日志文件保存格式为 W3C, 只记录日期、时间、客户端 IP 地址、用户名、方法)；

2. 关闭 IIS 的 WebDAV 功能增强网站的安全性。

3. 关闭 FTP 匿名访问；

4. 为了解决 IIS 短文件名漏洞，设置 URL 序列为～；

5. 设置网站最大并发连接数为 10。

# A-10 WEB 安全加固（Windows）

1. 为了防止 web 中.mdb 数据库文件非法下载，请对 Web 配置文件进行安全加固；

2. 限制目录执行权限，对 picture 和 upload 目录设置执行权限为无；

3. 开启 IIS 的日志审计记录 (日志文件保存格式为 W3C, 只记录日期、时间、客户端 IP 地址、用户名、方法)。

# A-11 Nginx 安全策略（Linux）

1. 禁止目录浏览和隐藏服务器版本和信息显示；

2. 限制 HTTP 请求方式，只允许 GET、HEAD、POST；

3. 设置客户端请求主体读取超时时间为 10；

4. 设置客户端请求头读取超时时间为 10；

5. 将 Nginx 服务降权，使用 www 用户启动服务
